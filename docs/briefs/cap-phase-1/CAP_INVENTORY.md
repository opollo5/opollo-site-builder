## 3. Role system pattern

**File**: `supabase/migrations/0070_platform_foundation.sql`

**Pattern**: `platform_users` table has `is_opollo_staff BOOLEAN NOT NULL DEFAULT false`. Supported by `is_opollo_staff()` SECURITY DEFINER function. Used in RLS policies and API gate code.

**`platform_company_role` enum** (also 0070): `admin`, `approver`, `editor`, `viewer`. This is for company-user memberships — NOT the right pattern for an Opollo-side operator role.

**Cap operator pattern** (following `is_opollo_staff`):
- Add `is_cap_operator BOOLEAN NOT NULL DEFAULT false` to `platform_users`
- Create `is_cap_operator()` SECURITY DEFINER function
- API gate checks `is_opollo_staff() OR is_cap_operator()`
