# Audits

Frozen-in-time analyses of architectural state at a specific date. Each
audit captures findings before a major workstream so future agents can
trace why decisions were made.

| Audit | Captures |
|---|---|
| [DESIGN_SYSTEM_2026-05-02](DESIGN_SYSTEM_2026-05-02.md) | Pre-DESIGN-SYSTEM-OVERHAUL Q1–Q8 audit + locked architecture decisions |

Post-workstream contracts (current behaviour) live separately in
`docs/architecture/`. Refer there for "what does the system do today" —
this directory is for "what state was it in when we made decision X".
